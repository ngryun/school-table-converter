# 엑셀 → 한글 표 변환기

rhwp WASM을 사용해서, 브라우저에서 엑셀 파일을 .hwp 표 문서로 변환하는 단일 정적 웹앱.
파일은 서버로 업로드하지 않고 브라우저 안에서 처리합니다.

## 구성

```
school-table-poc/
├── index.html        UI
├── app.js            SheetJS + rhwp WASM 호출 로직
├── lib/
│   ├── sheetjs/xlsx.full.min.js    SheetJS 0.18.5
│   └── rhwp/                        rhwp pkg/ 복사본 (rhwp.js / rhwp_bg.wasm / rhwp.d.ts)
└── README.md
```

## 로컬 실행

브라우저 보안상 `file://`로는 WASM 로딩이 막힙니다. `npm start`로 내장 Node 정적 서버를 띄워야 합니다.

```bash
cd school-table-poc
npm start
# 브라우저에서 http://localhost:8081/
```

## 웹 배포

이 프로젝트는 빌드 과정이 필요 없는 정적 사이트입니다. 배포 대상의 publish/output directory를 프로젝트 루트(`.`)로 지정하면 됩니다.
오픈소스 라이선스 표기는 `THIRD_PARTY_NOTICES.md`에 정리되어 있습니다.

### Vercel

1. 이 폴더를 GitHub 저장소로 올립니다.
2. Vercel에서 저장소를 Import합니다.
3. Framework Preset은 `Other`, Build Command는 비워두고, Output Directory는 `.`로 둡니다.
4. `vercel.json`이 WASM MIME 타입과 캐시 헤더를 지정합니다.

### Netlify

1. 이 폴더를 GitHub 저장소로 올립니다.
2. Netlify에서 저장소를 Import합니다.
3. Build command는 비워두고, Publish directory는 `.`로 지정합니다.
4. `_headers`가 WASM MIME 타입과 캐시 헤더를 지정합니다.

### GitHub Pages

1. 이 폴더를 GitHub 저장소로 올립니다.
2. 저장소 Settings → Pages로 이동합니다.
3. Source를 `GitHub Actions`로 선택합니다.
4. `main` 브랜치에 push하면 `.github/workflows/pages.yml`이 프로젝트 루트를 그대로 배포합니다.
5. GitHub Pages에서 WASM이 정상 로드되는지 셀프 테스트 버튼으로 확인합니다.

GitHub Pages는 정적 파일만 제공하므로 이 앱과 잘 맞습니다. `.nojekyll`은 GitHub Pages가 Jekyll 처리를 건너뛰도록 해 정적 파일을 있는 그대로 배포하게 합니다.

## 검증 순서

1. **셀프 테스트 버튼** 클릭 → `selftest.hwp` 다운로드.
   - 한컴 한글에서 열어보고 다음 확인:
     - 3×3 표가 보이는가
     - 첫 행이 (0,0)-(0,2) 병합돼 있는가
     - (0,0)에 "안녕하세요", (1,1)에 "한글 셀"이 들어가 있는가
2. **엑셀 파일 선택** → "선택한 엑셀 → hwp 변환" 클릭 → `<원본이름>.hwp` 다운로드.
   - 첫 행/제목 행, 머리 행, 첫 열에 절제된 색상 테마가 적용됩니다.
   - 실제 데이터 앞의 빈 행/열은 자동으로 잘라냅니다.
   - 병합 셀은 HWP 네이티브 표 API로 반영됩니다.
   - 열이 많은 표는 자동으로 가로 용지를 사용합니다.
   - `교육과정편제표` 양식은 `B2`를 문서 제목으로, `B5:O(B열 마지막 데이터행)`을 표로 자동 인식합니다.
   - `학점`, `1-1`처럼 숫자 위주의 학기별 학점 열은 좁게 배치합니다.
   - 학점/학기별 학점 열의 `0` 값은 빈칸으로 표시합니다.
   - `학기별 과목수 최소/최대선택` 안내는 `선택그룹명` 셀에 붙이고 안내 전용 열은 제거합니다.
   - 로그 영역에 단계별 진행 상황과 실패 건수가 표시됩니다.

## 알려진 한계

- 원본 xlsx의 모든 셀별 색상/폰트 서식을 그대로 복제하지는 않습니다. 대신 시간표용 고정 테마를 입힙니다.
- rhwp 네이티브 API가 직접 반영하지 못하는 셀 배경색은 `.hwp` 내 `BorderFill` 레코드를 후처리해 적용합니다.

## 로직 메모

- xlsx/xls/xlsm을 읽어 값 셀, 병합 범위, 열 너비를 추출한다.
- 교육과정편제표 양식은 시트명 또는 `B2` 제목으로 감지하고, 제목은 표 밖 본문에 한 번만 배치하며 `B5:O` 고정 범위를 표로 사용한다.
- 값이 있는 영역과 그 영역에 닿는 병합 범위만 남겨 앞쪽 빈 행/열을 제거한다.
- 한컴 한글 렌더링 안정성을 위해 `createTable`로 HWP 네이티브 표를 만든다.
- 헤더명을 기준으로 과목/분류 열은 넓게, 학점/학기별 숫자 열은 좁게 열폭을 재계산한다.
- 학점/학기별 학점 열의 0 값은 텍스트 삽입 단계에서 생략한다.
- 학기별 과목수 최소/최대선택 안내 문구는 선택그룹명 열로 이동하고, 안내 전용 열이면 삭제한다.
- 헤더는 전용 셀 스타일로 흰색/굵게 처리하고, 본문은 낮은 채도와 얇은 그리드 위주로 정리한다.
- 텍스트/폰트/굵기/정렬을 적용하고 병합을 마지막에 처리한 뒤, 셀 배경색은 HWP `DocInfo` 후처리로 보정한다.
